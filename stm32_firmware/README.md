# STM32 Firmware для WatchDog Robot

Полная прошивка для микроконтроллера STM32F407, управляющего моторами и обеспечивающего связь с ROS2.

## Функции

- ✅ Управление двумя моторами через PWM
- ✅ Чтение энкодеров для одометрии
- ✅ Связь с ROS2 через UART
- ✅ Обработка команд движения
- ✅ Отправка данных энкодеров

## Структура проекта

```
stm32_firmware/
├── src/
│   ├── main.c              # Основной файл с главным циклом
│   ├── main.h               # Заголовочный файл
│   ├── motor_control.c/h    # Управление моторами
│   ├── encoder_reader.c/h   # Чтение энкодеров
│   └── communication.c/h    # Протокол связи с ROS2
├── docs/
│   └── STM32_SETUP.md       # Инструкции по настройке
├── STM32CubeMX_Configuration.md  # Конфигурация CubeMX
└── README.md                # Этот файл
```

## Требования

- STM32CubeIDE или STM32CubeMX + STM32CubeIDE
- STM32F407 (или совместимая модель)
- ST-Link для прошивки
- UART для связи с ROS2

## Быстрый старт

### 1. Настройка через STM32CubeMX

1. Откройте STM32CubeMX
2. Создайте новый проект для STM32F407
3. Следуйте инструкциям в `STM32CubeMX_Configuration.md`

**Основные настройки:**
- UART1 для связи с ROS2
- TIM2/TIM3 для PWM моторов
- TIM4/TIM5 для энкодеров
- GPIO для управления направлением

### 2. Добавление пользовательского кода

После генерации кода в STM32CubeMX:

1. Скопируйте файлы из `src/` в ваш проект:
   - `motor_control.c/h`
   - `encoder_reader.c/h`
   - `communication.c/h`

2. Обновите `main.c`:
   - Добавьте вызовы инициализации
   - Добавьте основной цикл из `main.c`

3. Добавьте заголовочные файлы в includes:
   ```c
   #include "motor_control.h"
   #include "encoder_reader.h"
   #include "communication.h"
   ```

### 3. Компиляция и прошивка

1. Соберите проект в STM32CubeIDE
2. Подключите ST-Link
3. Загрузите прошивку на микроконтроллер

## Подключение

### Схема подключения

**Левый мотор:**
```
Драйвер DRV8833:
  IN1 → STM32 PA0 (GPIO)
  IN2 → STM32 PA1 (GPIO)
  EN  → STM32 PA2 (TIM2_CH1, PWM)
  
Энкодер:
  A → STM32 PB6 (TIM4_CH1)
  B → STM32 PB7 (TIM4_CH2)
```

**Правый мотор:**
```
Драйвер DRV8833:
  IN1 → STM32 PA3 (GPIO)
  IN2 → STM32 PA4 (GPIO)
  EN  → STM32 PA5 (TIM3_CH1, PWM)
  
Энкодер:
  A → STM32 PC6 (TIM8_CH1) или альтернативный пин
  B → STM32 PC7 (TIM8_CH2) или альтернативный пин
```

**UART (ROS2):**
```
STM32          Raspberry Pi
─────────────────────────────
PA9 (TX)  →    UART RX
PA10 (RX) ←    UART TX
GND       →    GND
```

## Тестирование

### 1. Проверка связи

```bash
# На ROS2 стороне
ros2 run watchdog_stm32_interface stm32_interface_node

# Проверьте статус
ros2 topic echo /stm32/status
```

### 2. Тест моторов

```bash
# Отправьте команду движения
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.5}, angular: {z: 0.0}}"

# Робот должен двигаться вперед
```

### 3. Проверка энкодеров

```bash
# Просмотр odometry данных
ros2 topic echo /odom

# Вручную покрутите колеса и проверьте изменение значений
```

## Калибровка

### Калибровка энкодеров

В `encoder_reader.c` настройте параметры:
- Количество тиков на оборот (зависит от энкодера)
- Направление вращения (если нужно инвертировать)

### Калибровка моторов

В `motor_control.c` настройте:
- `WHEEL_BASE` - расстояние между колесами
- `WHEEL_RADIUS` - радиус колеса
- `MAX_PWM_VALUE` - максимальное значение PWM
- Максимальные RPM колеса

## Отладка

### Использование ST-Link

```bash
# Подключите ST-Link к STM32
# В STM32CubeIDE: Run → Debug

# Или через openocd
openocd -f interface/stlink.cfg -f target/stm32f4x.cfg
```

### Логирование через UART

Можно добавить printf через UART для отладки:
```c
// В main.c
printf("Motor control initialized\r\n");
```

### Проверка значений

В отладчике можно проверить:
- Значения PWM: `__HAL_TIM_GET_COMPARE(&htim2, TIM_CHANNEL_1)`
- Значения энкодеров: `__HAL_TIM_GET_COUNTER(&htim4)`
- Состояние UART: `huart1.RxState`

## Параметры конфигурации

### motor_control.c

```c
#define WHEEL_BASE 0.25f      // Расстояние между колесами (м)
#define WHEEL_RADIUS 0.05f    // Радиус колеса (м)
#define MAX_PWM_VALUE 1000   // Максимальное значение PWM
```

### encoder_reader.c

Количество тиков на оборот зависит от энкодера. Обычно:
- 360 тиков/оборот для простых энкодеров
- 1000+ для высокоточных

## Протокол связи

Полное описание протокола см. в `docs/STM32_PROTOCOL.md`.

**Основные команды:**
- `0x01` - Команда движения (linear_x, angular_z)
- `0x02` - Запрос состояния
- `0x12` - Ответ с данными энкодеров

## Решение проблем

### Проблема: "Моторы не крутятся"

1. Проверьте питание драйвера (12V)
2. Проверьте подключение GPIO
3. Проверьте PWM сигнал осциллографом
4. Проверьте логику управления направлением

### Проблема: "Энкодеры не работают"

1. Проверьте подключение к TIM пинам
2. Убедитесь, что таймер в режиме Encoder
3. Проверьте уровень сигналов (3.3V)
4. Проверьте инверсию каналов если нужно

### Проблема: "Нет связи с ROS2"

1. Проверьте UART подключение
2. Проверьте скорость передачи (115200)
3. Проверьте уровни (3.3V или 5V)
4. Проверьте права доступа к порту на ROS2 стороне

### Проблема: "Неправильная одометрия"

1. Проверьте количество тиков на оборот
2. Проверьте радиус колеса
3. Проверьте калибровку WHEEL_BASE
4. Проверьте направление энкодеров (может быть инвертировано)

## Расширение функционала

### Добавление IMU

Можно добавить модуль для чтения IMU (MPU6050, MPU9250) через I2C:
```c
// В main.c
MX_I2C1_Init();
IMU_Init();
IMU_ReadData(&imu_data);
```

### Добавление датчиков столкновения

GPIO для чтения датчиков:
```c
if (HAL_GPIO_ReadPin(GPIOB, GPIO_PIN_12) == GPIO_PIN_RESET) {
    // Обнаружено столкновение
    MotorControl_Stop();
}
```

## Производительность

- **Частота обновления команд:** До 50 Гц
- **Частота отправки энкодеров:** 50 Гц
- **Задержка обработки:** < 10 мс
- **Точность одометрии:** Зависит от энкодеров (обычно ±2-5%)

## Примечания

- Код рассчитан на STM32F407, но может быть адаптирован для других моделей
- Все пины настраиваются через STM32CubeMX
- При изменении пинов обновите константы в коде
- Для продакшена рекомендуется использовать DMA для UART
