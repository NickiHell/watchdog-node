# Настройка STM32 для WatchDog Robot

## Требования

- STM32CubeIDE или STM32CubeMX
- STM32F407 (или другая совместимая модель)
- UART для связи с ROS2
- Таймеры для PWM управления моторами
- Таймеры для чтения энкодеров

## Конфигурация через STM32CubeMX

### 1. UART (USART1)

- **Режим:** Asynchronous
- **Baudrate:** 115200
- **Data Bits:** 8
- **Parity:** None
- **Stop Bits:** 1
- **RX Pin:** PA10 (UART1_RX)
- **TX Pin:** PA9 (UART1_TX)

### 2. Таймеры для PWM (TIM2)

- **Mode:** PWM Generation
- **Channel 1:** PA0 (для левого мотора - IN1)
- **Channel 2:** PA1 (для левого мотора - IN2)
- **Channel 3:** PA2 (для левого мотора - EN)
- **Channel 4:** PA3 (для правого мотора - IN1)
- **Prescaler:** Настроить для частоты ~20kHz
- **Counter Period:** 999 (для разрешения 1000 шагов)

### 3. Таймеры для энкодеров

**Левый энкодер (TIM3):**
- **Mode:** Encoder Mode
- **Channel 1:** PB6 (TIM3_CH1)
- **Channel 2:** PB7 (TIM3_CH2)

**Правый энкодер (TIM4):**
- **Mode:** Encoder Mode
- **Channel 1:** PB8 (TIM4_CH1)
- **Channel 2:** PB9 (TIM4_CH2)

### 4. GPIO для управления направлением моторов

- **PA4:** Правый мотор - IN1
- **PA5:** Правый мотор - IN2
- **PA6:** Правый мотор - EN

(Левая сторона уже настроена через TIM2)

## Порядок разработки

1. **Создать проект в STM32CubeMX**
   - Выбрать модель микроконтроллера
   - Настроить UART, таймеры, GPIO

2. **Сгенерировать код**
   - Сохранить проект
   - Сгенерировать код для STM32CubeIDE

3. **Добавить файлы протокола**
   - Скопировать `communication.h` и `communication.c`
   - Адаптировать под вашу конфигурацию UART

4. **Реализовать модули управления**
   - `motor_control.c` - управление моторами через PWM
   - `encoder_reader.c` - чтение энкодеров через таймеры

5. **Интегрировать в main.c**
   - Добавить вызовы инициализации
   - Реализовать основной цикл обработки

## Тестирование

### Тест связи

1. Подключить STM32 к компьютеру через UART
2. Запустить `stm32_interface_node` на ROS2
3. Отправить команду движения:
```bash
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.5}, angular: {z: 0.0}}"
```

4. Проверить odometry:
```bash
ros2 topic echo /odom
```

### Тест энкодеров

1. Вручную покрутить колеса
2. Проверить изменение значений в топике `/odom`

## Отладка

- Использовать STM32 ST-Link для отладки
- Добавить логирование через UART (отдельный канал или условная компиляция)
- Использовать осциллограф для проверки PWM сигналов

## Замечания

- **Прерывания:** Рекомендуется использовать прерывания UART вместо polling для лучшей производительности
- **DMA:** Для высокоскоростной передачи можно использовать DMA
- **Защита:** Добавить таймауты и проверки валидности данных
- **Многопоточность:** Если используется FreeRTOS, нужно добавить мьютексы для защиты общих данных
