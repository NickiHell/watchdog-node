# Настройка лидара для WatchDog Robot

Подробное руководство по выбору, подключению и настройке лидара.

## Выбор лидара

### Рекомендации

**Лучший выбор для начала: RPLidar A1**
- Хорошо документирован
- Отличная поддержка в ROS2
- Доступная цена (~$100-150)
- Достаточно для навигации в помещении

**Оптимальный вариант: RPLidar A2**
- Увеличенная дальность (до 16м)
- Лучшая точность
- Хорошее соотношение цена/качество

**Для серьезных проектов: RPLidar A3**
- Максимальная дальность (до 25м)
- Высокая частота сканирования

## Альтернативы RPLidar

### Бюджетные варианты

1. **Лидары от пылесосов** (Xiaomi MOP3, Dreame и т.д.)
   - Требует реверс-инжиниринг протокола
   - Может быть сложно найти документацию
   - Дешево (если уже есть)
   - ВНИМАНИЕ: Используйте generic режим для экспериментов

2. **YDLidar**
   - Похож на RPLidar
   - Хорошая поддержка
   - ВНИМАНИЕ: Может потребоваться адаптация драйвера

### Профессиональные варианты

1. **Hokuyo URG-04LX**
   - Очень качественный
   - Дорогой (~$500+)
   - Отличная документация

2. **Velodyne** (для серьезных проектов)
   - 3D лидар
   - Очень дорогой
   - Профессиональное качество

## Подключение RPLidar

### Шаг 1: Физическое подключение

1. Подключите лидар через USB к Raspberry Pi
2. Убедитесь в достаточном питании (обычно 5V, проверьте спецификацию)
3. Проверьте светодиоды на лидаре (должны гореть)

### Шаг 2: Проверка подключения

```bash
# Проверьте, что устройство обнаружено
ls -l /dev/ttyUSB* /dev/ttyACM*

# Должно появиться что-то вроде:
# /dev/ttyUSB0
```

### Шаг 3: Настройка прав доступа

```bash
# Добавьте пользователя в группу dialout
sudo usermod -a -G dialout $USER

# Перелогиньтесь или выполните:
newgrp dialout

# Проверьте права
ls -l /dev/ttyUSB0
# Должно быть: crw-rw-rw- или crw-rw---- dialout
```

Если не работает, можно временно:
```bash
sudo chmod 666 /dev/ttyUSB0
```

### Шаг 4: Тестирование подключения

```bash
# Попробуйте прочитать данные (для отладки)
cat /dev/ttyUSB0 | hexdump -C | head -20

# Или через Python
python3 -c "import serial; s = serial.Serial('/dev/ttyUSB0', 115200); print(s.read(100))"
```

## Настройка ROS2 узла

### Базовая конфигурация

```bash
ros2 run watchdog_lidar lidar_node --ros-args \
  -p lidar.type:=rplidar \
  -p lidar.port:=/dev/ttyUSB0 \
  -p lidar.baudrate:=115200 \
  -p lidar.scan_rate:=10.0
```

### Для разных моделей RPLidar

**RPLidar A1:**
```bash
-p lidar.baudrate:=115200
```

**RPLidar A2:**
```bash
-p lidar.baudrate:=115200
```

**RPLidar A3:**
```bash
-p lidar.baudrate:=256000
```

### Проверка работы

```bash
# Терминал 1: Запустите узел
ros2 run watchdog_lidar lidar_node

# Терминал 2: Проверьте данные
ros2 topic echo /sensor/lidar/scan

# Терминал 3: Проверьте статус
ros2 topic echo /lidar/status
```

### Визуализация в RViz2

```bash
# Запустите RViz2
rviz2

# Добавьте LaserScan:
# Fixed Frame: lidar_frame
# Add → By topic → /sensor/lidar/scan
```

## Работа с MOP3 лидаром

Если у вас есть лидар от Xiaomi MOP3:

### Шаг 1: Подключение

1. Найдите USB-UART адаптер (CP2102, CH340 и т.д.)
2. Подключите лидар через адаптер к Raspberry Pi
3. Определите порт: `ls -l /dev/ttyUSB*`

### Шаг 2: Экспериментальный режим

```bash
ros2 run watchdog_lidar lidar_node --ros-args \
  -p lidar.type:=generic \
  -p lidar.port:=/dev/ttyUSB0 \
  -p lidar.baudrate:=115200
```

### Шаг 3: Анализ данных

Generic драйвер пытается автоматически определить формат. Если не работает:

1. Запишите сырые данные:
   ```python
   import serial
   s = serial.Serial('/dev/ttyUSB0', 115200)
   with open('lidar_data.bin', 'wb') as f:
       f.write(s.read(10000))
   ```

2. Проанализируйте hex дамп:
   ```bash
   hexdump -C lidar_data.bin | head -50
   ```

3. Ищите паттерны:
   - Синхронизационные байты (0xAA, 0x55 и т.д.)
   - Заголовки пакетов
   - Расстояния (обычно 2-4 байта)

### Шаг 4: Создание специфичного драйвера

После анализа протокола можно создать драйвер для MOP3:

```python
class MOP3LidarDriver(LidarDriver):
    # Реализуйте методы на основе найденного протокола
    pass
```

## Решение проблем

### Проблема: "Permission denied"

```bash
# Решение 1: Группа dialout
sudo usermod -a -G dialout $USER
# Перелогиньтесь

# Решение 2: Временные права
sudo chmod 666 /dev/ttyUSB0
```

### Проблема: "Device not found"

1. Проверьте подключение кабеля
2. Проверьте питание лидара
3. Попробуйте другой USB порт
4. Проверьте: `dmesg | tail` для системных сообщений

### Проблема: "Нет данных"

1. Проверьте скорость передачи (baudrate)
2. Для RPLidar: попробуйте 115200 и 256000
3. Проверьте логи: `ros2 topic echo /lidar/status`
4. Убедитесь, что лидар запущен и вращается

### Проблема: "Неправильные расстояния"

1. Проверьте range_min и range_max параметры
2. Убедитесь в правильном типе лидара
3. Проверьте калибровку лидара (если требуется)

## Производительность

### Оптимизация для Raspberry Pi

```yaml
lidar:
  scan_rate: 5.0  # Снизить частоту
  range_max: 8.0  # Ограничить дальность
```

### Для мощных систем

```yaml
lidar:
  scan_rate: 15.0  # Увеличить частоту
  range_max: 12.0  # Полная дальность
```
