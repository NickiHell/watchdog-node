# Протокол связи STM32 ↔ ROS2

Детальное описание протокола обмена данными между ROS2 узлом и STM32 микроконтроллером.

## Формат сообщений

Все сообщения имеют следующую структуру:

```
[Заголовок (2 байта)] [Тип (1 байт)] [Данные (N байт)] [Контрольная сумма (1 байт)]
```

### Заголовок

Всегда: `[0xAA, 0x55]`

### Контрольная сумма

XOR всех байт сообщения (кроме самой контрольной суммы):
```python
checksum = 0
for byte in packet_without_checksum:
    checksum ^= byte
```

## Команды от ROS2 к STM32

### 1. Команда движения (CMD_MOVEMENT = 0x01)

**Назначение:** Управление движением робота (cmd_vel).

**Формат:**
```
Байт 0-1:    Заголовок [0xAA, 0x55]
Байт 2:      Тип команды [0x01]
Байт 3-6:    linear.x (float, little-endian, 32-bit)
Байт 7-10:   angular.z (float, little-endian, 32-bit)
Байт 11:     Контрольная сумма
```

**Размер пакета:** 12 байт

**Пример:**
```
0xAA 0x55 0x01 0x00 0x00 0x00 0x3F 0x00 0x00 0x80 0x3F 0xXX
              ^^^  ^^^^^^^^^^^^^^^^  ^^^^^^^^^^^^^^^^  ^^^
            тип    linear.x=0.5      angular.z=1.0      chk
```

### 2. Запрос состояния (CMD_STATUS_REQUEST = 0x02)

**Назначение:** Запрос текущего состояния или данных энкодеров.

**Формат:**
```
Байт 0-1:    Заголовок [0xAA, 0x55]
Байт 2:      Тип команды [0x02]
Байт 3:      Контрольная сумма
```

**Размер пакета:** 4 байта

**Пример:**
```
0xAA 0x55 0x02 0xXX
```

**Ответ:** STM32 должен отправить данные энкодеров (RESP_ENCODER_DATA).

### 3. Установка параметров (CMD_PARAM_SET = 0x03)

**Назначение:** Установка параметров (резерв для будущего использования).

**Формат:**
```
Байт 0-1:    Заголовок [0xAA, 0x55]
Байт 2:      Тип команды [0x03]
Байт 3-N:    Параметры (формат TBD)
Байт N+1:    Контрольная сумма
```

## Ответы от STM32 к ROS2

### 1. Успех (RESP_SUCCESS = 0x11)

**Назначение:** Подтверждение успешного выполнения команды.

**Формат:**
```
Байт 0-1:    Заголовок [0xAA, 0x55]
Байт 2:      Тип ответа [0x11]
Байт 3:      Контрольная сумма
```

**Размер пакета:** 4 байта

**Пример:**
```
0xAA 0x55 0x11 0xXX
```

### 2. Ошибка (RESP_ERROR = 0x10)

**Назначение:** Сообщение об ошибке.

**Формат:**
```
Байт 0-1:    Заголовок [0xAA, 0x55]
Байт 2:      Тип ответа [0x10]
Байт 3:      Код ошибки (uint8)
Байт 4:      Контрольная сумма
```

**Размер пакета:** 5 байт

**Коды ошибок:**
- `0x01`: Неверная контрольная сумма
- `0x02`: Переполнение буфера
- `0x03`: Неизвестная команда
- `0x04`: Ошибка параметров

**Пример:**
```
0xAA 0x55 0x10 0x01 0xXX
              ^^^  ^^^
            тип    код ошибки
```

### 3. Данные энкодеров (RESP_ENCODER_DATA = 0x12)

**Назначение:** Передача данных энкодеров для вычисления odometry.

**Формат:**
```
Байт 0-1:    Заголовок [0xAA, 0x55]
Байт 2:      Тип ответа [0x12]
Байт 3-6:    encoder_left (int32, little-endian, знаковое)
Байт 7-10:   encoder_right (int32, little-endian, знаковое)
Байт 11-14:  timestamp_ms (uint32, little-endian, миллисекунды)
Байт 15:     Контрольная сумма
```

**Размер пакета:** 16 байт

**Пример:**
```
0xAA 0x55 0x12 0x64 0x00 0x00 0x00 0xC8 0x00 0x00 0x00 0x12 0x34 0x56 0x78 0xXX
              ^^^  ^^^^^^^^^^^^^^^^  ^^^^^^^^^^^^^^^^  ^^^^^^^^^^^^^^^^  ^^^
            тип    left=100          right=200         timestamp         chk
```

## Последовательность взаимодействия

### Типичный цикл работы

1. **ROS2 → STM32:** Команда движения (CMD_MOVEMENT)
2. **STM32 → ROS2:** Подтверждение (RESP_SUCCESS)
3. **ROS2 → STM32:** Запрос состояния (CMD_STATUS_REQUEST)
4. **STM32 → ROS2:** Данные энкодеров (RESP_ENCODER_DATA)
5. Повторяется с частотой ~50 Гц

### Обработка ошибок

Если STM32 получает неверный пакет:
- Отправляет RESP_ERROR с соответствующим кодом
- Игнорирует команду
- Ждет следующую команду

Если ROS2 получает неверный ответ:
- Логирует предупреждение
- Продолжает работу
- Отправляет повторный запрос

## Реализация на стороне STM32

### Пример структуры данных

```c
typedef struct {
    uint8_t header[2];
    uint8_t type;
    union {
        struct {
            float linear_x;
            float angular_z;
        } movement;
        uint8_t error_code;
        struct {
            int32_t encoder_left;
            int32_t encoder_right;
            uint32_t timestamp_ms;
        } encoders;
    } data;
    uint8_t checksum;
} Packet;
```

### Алгоритм приема

1. Искать заголовок `0xAA 0x55`
2. Прочитать тип команды
3. Определить ожидаемую длину пакета
4. Дождаться полного пакета
5. Проверить контрольную сумму
6. Обработать команду

### Алгоритм отправки

1. Сформировать заголовок
2. Добавить тип ответа
3. Добавить данные
4. Вычислить контрольную сумму
5. Отправить через UART

## Тестирование

### Ручной тест (Python)

```python
import serial
from watchdog_stm32_interface.protocol import STM32Protocol

# Открыть порт
ser = serial.Serial('/dev/ttyACM0', 115200)

# Отправить команду движения
cmd = STM32Protocol.encode_movement_command(0.5, 0.0)
ser.write(cmd)

# Прочитать ответ
response = ser.read(16)
result = STM32Protocol.decode_response(response)
print(result)
```

### Автоматический тест (ROS2)

```bash
# Терминал 1: Запустить узел
ros2 run watchdog_stm32_interface stm32_interface_node

# Терминал 2: Отправить команду
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.5}, angular: {z: 0.0}}"

# Терминал 3: Проверить odometry
ros2 topic echo /odom
```

## Примечания

- Все числа в формате little-endian
- Float - IEEE 754 single precision (32-bit)
- int32 - знаковое 32-битное число
- uint32 - беззнаковое 32-битное число
- Таймауты: ROS2 ожидает ответ в течение 100мс
- Частота обновления: рекомендуется 50 Гц для энкодеров
