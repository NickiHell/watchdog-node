# Зачем нужен Raspberry Pi, если есть STM32 Nucleo?

## Краткий ответ

**STM32 Nucleo** и **Raspberry Pi** выполняют **разные задачи** и **не могут заменить друг друга**:

- **STM32 Nucleo** = низкоуровневое управление моторами (железо)
- **Raspberry Pi** = высокоуровневая логика, ROS2, обработка данных (мозг)

## Детальное сравнение

### STM32 Nucleo (F411RE) - "Низкоуровневый контроллер"

**Что делает:**
- ✅ Управление моторами через PWM (частота 20kHz, точное время)
- ✅ Чтение энкодеров в реальном времени (50 Гц)
- ✅ Обработка команд движения (linear_x, angular_z)
- ✅ Отправка данных одометрии обратно

**Характеристики:**
- Процессор: ARM Cortex-M4, 100MHz
- Память: 512KB Flash, 128KB RAM
- Операционная система: **НЕТ** (bare metal)
- Языки: C/C++ (без стандартной библиотеки)
- Реальное время: ✅ Да, гарантированное

**Что НЕ может делать:**
- ❌ Запустить ROS2 (требует Linux)
- ❌ Обработать данные лидара (SLAM, навигация)
- ❌ Распознавание лиц (ML модели требуют много памяти)
- ❌ Обработка изображений с камеры
- ❌ Голосовое взаимодействие (STT/TTS)
- ❌ Работа с USB устройствами (камера, лидар)
- ❌ Работа с файловой системой
- ❌ Сетевое взаимодействие

### Raspberry Pi - "Высокоуровневый компьютер"

**Что делает:**
- ✅ Запуск ROS2 (Linux операционная система)
- ✅ Обработка данных с лидара (SLAM, построение карты)
- ✅ Обработка изображений с камеры
- ✅ Распознавание лиц (ML модели)
- ✅ Голосовое взаимодействие (STT/TTS)
- ✅ Навигация и планирование пути
- ✅ Координация всех модулей
- ✅ Связь с STM32 через UART

**Характеристики:**
- Процессор: ARM Cortex-A76, 2.4GHz (4-8 ядер)
- Память: 4-8GB RAM
- Операционная система: **Linux** (Ubuntu/Raspberry Pi OS)
- Языки: Python, C++ (с полной стандартной библиотекой)
- Реальное время: ⚠️ Нет гарантий (не RTOS)

## Примеры задач

### Задача 1: "Поехать к точке X"

**Raspberry Pi:**
1. Получает данные лидара
2. Строит карту (SLAM)
3. Планирует путь к точке X
4. Вычисляет команды движения (linear_x, angular_z)
5. Отправляет команды STM32 через UART

**STM32:**
1. Получает команды от Raspberry Pi
2. Конвертирует в PWM сигналы для моторов
3. Управляет моторами в реальном времени
4. Читает энкодеры
5. Отправляет данные одометрии обратно

**Без Raspberry Pi:** STM32 не знает, где находится робот и куда ехать  
**Без STM32:** Raspberry Pi не может управлять моторами напрямую

### Задача 2: "Распознать лицо и поприветствовать"

**Raspberry Pi:**
1. Получает изображение с камеры
2. Запускает модель распознавания лиц
3. Определяет знакомое/незнакомое лицо
4. Синтезирует речь "Привет, Иван!"
5. Воспроизводит через динамик

**STM32:**
- Не участвует (это задача только Raspberry Pi)

**Без Raspberry Pi:** Невозможно (STM32 не может обработать изображения)

## Можно ли обойтись без Raspberry Pi?

### ❌ НЕТ, если нужен полный функционал:

- **ROS2:** Требует Linux, минимум 512MB RAM, много вычислительной мощности
- **SLAM:** Требует обработки больших массивов данных лидара
- **ML модели:** Распознавание лиц требует 100+ MB RAM и GPU/CPU для инференса
- **Обработка изображений:** OpenCV, Python библиотеки
- **Голосовое взаимодействие:** STT/TTS модели требуют много ресурсов

### ✅ МОЖНО, но только для простых задач:

Если упростить проект до:
- Простое движение вперед/назад/поворот
- Без навигации
- Без распознавания лиц
- Без голосового взаимодействия

Тогда можно использовать только STM32, но это будет **другой проект** (не WatchDog Robot).

## Альтернативы Raspberry Pi

Если Raspberry Pi дорого или недоступен:

1. **NVIDIA Jetson Nano** (~$150)
   - Больше производительности
   - GPU для ML
   - Дороже

2. **Intel NUC** (~$200-300)
   - x86 архитектура
   - Больше производительности
   - Дороже

3. **Orange Pi, Banana Pi** (~$40-60)
   - Дешевле
   - Может быть проблемы с драйверами
   - Меньше поддержки

4. **ПК/ноутбук** (если робот стационарный)
   - Максимальная производительность
   - Не мобильный
   - Подключение через USB/UART

## Вывод

**STM32 Nucleo** и **Raspberry Pi** - это **дополняющие друг друга** компоненты:

- **STM32** = низкоуровневое управление, реальное время, железо
- **Raspberry Pi** = высокоуровневая логика, ROS2, обработка данных, ML

Они работают **вместе**, каждый выполняет свою задачу. Это стандартная архитектура роботов (например, Turtlebot, Roomba и другие используют похожую схему).

## Схема коммуникации

```
Камера (USB) ──┐
                │
Лидар (USB) ────┼──→ Raspberry Pi ──UART──→ STM32 ──→ Моторы
                │      (ROS2)              (PWM)
Микрофон (USB) ─┘
```

**Каждое устройство подключается к Raspberry Pi**, а Raspberry Pi управляет STM32, который управляет моторами.
