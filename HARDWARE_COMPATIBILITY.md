# Анализ совместимости выбранных компонентов

## Выбранные компоненты

### ✅ 1. RPLiDAR ToF C1
**Статус:** ⚠️ Требует проверки совместимости

**Проблема:** Проект поддерживает RPLidar A1/A2/A3, но RPLiDAR ToF C1 может использовать другой протокол.

**Решение:**
- Если ToF C1 использует тот же протокол, что и A1/A2/A3 - будет работать с текущим драйвером
- Если протокол отличается - нужно использовать generic режим или создать адаптер

**Действия:**
1. Проверить документацию ToF C1 на совместимость с протоколом RPLidar
2. Попробовать подключить с типом `rplidar` в конфигурации
3. Если не работает, использовать `generic` режим и адаптировать драйвер

### ✅ 2. Logitech Веб-камера
**Статус:** ✅ Полностью совместимо

Проект поддерживает USB камеры через OpenCV. Просто подключите камеру и настройте в `camera_config.yaml`.

### ✅ 3. DRV8833 Драйвер моторов
**Статус:** ✅ Полностью совместимо

Драйвер DRV8833 полностью поддерживается проектом. Код уже настроен для работы с этим драйвером.

**Примечание:** Нужно **2 штуки** (по одному на каждый мотор).

### ⚠️ 4. NUCLEO-F411RE
**Статус:** ⚠️ Требует небольших доработок

Проект разработан для STM32F407, но F411RE очень похож и совместим с минимальными изменениями.

**Различия:**
- F411: 168MHz, 512KB Flash, 128KB RAM
- F407: 168MHz, 1MB Flash, 192KB RAM

**Действия:**
1. В STM32CubeMX выбрать STM32F411RE вместо F407
2. Проверить, что все пины доступны (F411 имеет меньше пинов, но для этого проекта достаточно)
3. Скомпилировать и прошить прошивку

## ❌ Недостающие компоненты

### 1. Основной компьютер (ROS2)
**Статус:** ✅ Beelink Mini PC - **уже выбрано** (отлично подходит!)

**Преимущества Beelink Mini PC:**
- ✅ Значительно мощнее Raspberry Pi (x86 архитектура)
- ✅ Больше RAM (обычно 8-16GB)
- ✅ Быстрее обработка данных (SLAM, ML, видео)
- ✅ SSD уже установлен (быстрее чем MicroSD)
- ✅ Больше USB портов
- ✅ Лучше для ROS2 и тяжелых задач

**Что нужно проверить:**
- Напряжение питания (обычно 12V или 19V) - нужно будет подобрать преобразователь
- Достаточно USB портов для камеры, лидара и других устройств

### 2. Моторы и энкодеры
**Статус:** ✅ Выбрано

**Моторы:**
- **25GA370** (2 шт) - мотор-редуктор постоянного тока, 12V
- Без встроенных энкодеров

**Энкодеры:**
- **E6B2-CWZ6C 1000P/R** (Omron) (2 шт) - инкрементальный энкодер
- Разрешение: 1000 P/R = 4000 counts/revolution (quadrature)
- Напряжение: 5V (требует делители для STM32 3.3V)

### 3. Источники питания
**Статус:** ✅ Частично выбрано

**Выбрано:**
- **LiPo 12V 10000mAh** - 12V (3S), разъем 5.5×2.1мм ✅

  - **Отлично! Хватит на 2-3 часа работы**
- **LM2596S 8-40V→5V** (2 шт) - для энкодеров
- **USB-UART CP2102** - для подключения лидара

## 📝 Примечания

- **DRV8833:** Убедитесь, что покупаете драйвер для моторов постоянного тока (не шаговых), хотя название говорит "шагового двигателя"
- **Моторы:** Критически важно выбрать моторы с встроенными энкодерами для точной навигации
- **Питание:** Убедитесь в достаточной емкости батареи для работы минимум 1-2 часа
- **Лидар:** RPLiDAR ToF C1 может потребовать адаптации драйвера, если протокол отличается
