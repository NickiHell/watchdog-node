# Security Policy

## Поддерживаемые версии

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | ✅                 |
| < 0.1   | ❌                 |

## Сообщение об уязвимости

Если вы обнаружили уязвимость безопасности, пожалуйста, сообщите об этом:

**Email:** nickihell@ya.ru

**Не публикуйте** информацию об уязвимости публично до тех пор, пока она не будет исправлена.

## Процесс

1. Отправьте описание уязвимости на указанный email
2. Ожидайте подтверждения получения (в течение 48 часов)
3. Мы рассмотрим уязвимость и подготовим исправление
4. После исправления мы опубликуем информацию об уязвимости и способах её устранения

## Типы уязвимостей

Мы принимаем сообщения о следующих типах уязвимостей:

- Небезопасная обработка данных
- Недостаточная валидация входных данных
- Утечки конфиденциальной информации
- Небезопасные сетевые протоколы
- Проблемы с правами доступа
- SQL/NoSQL инъекции (если применимо)
- XSS (если применимо)
- CSRF (если применимо)

## Меры безопасности

### Валидация входных данных

Все входные данные должны быть валидированы:

```python
from watchdog_common.config_validator import ConfigValidator

# Валидация конфигурации
config = ConfigValidator.load_and_validate('config/robot_config.yaml')
```

### Ограничение доступа

- ROS2 топики с конфиденциальными данными должны использовать соответствующие QoS настройки
- Используйте ROS2 security features для production окружений

### Безопасность протокола STM32

- Все команды валидируются на стороне STM32
- Проверка контрольной суммы обязательна
- Ограничение диапазонов команд движения

### Хранение данных

- База данных лиц хранится локально
- Эмбеддинги не содержат персональных данных
- Логи не должны содержать конфиденциальную информацию

## Рекомендации

1. **Не храните пароли и ключи в коде**
2. **Используйте переменные окружения для чувствительных данных**
3. **Регулярно обновляйте зависимости**
4. **Используйте HTTPS для удаленного доступа (если применимо)**
5. **Ограничьте сетевой доступ к ROS2 узлам**

## Обновления безопасности

Все исправления безопасности будут опубликованы в:
- Release notes
- Security advisories
- Через email для подписчиков
