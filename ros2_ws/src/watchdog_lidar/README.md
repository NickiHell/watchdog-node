# watchdog_lidar

ROS2 пакет для работы с лидарами. Поддерживает RPLidar и универсальный режим для экспериментов с другими лидарами (например, MOP3).

## Поддерживаемые лидары

### RPLidar (рекомендуется)

**Модели:**
- **RPLidar A1** - до 8 метров, 2-10 Гц
- **RPLidar A2** - до 16 метров, 5-15 Гц
- **RPLidar A3** - до 25 метров, 5-15 Гц

**Преимущества:**
- Хорошо документирован
- Популярный в ROS сообществе
- Надежный протокол
- Доступная цена (~$100-200)

**Где купить:**
- Официальный сайт SLAMTEC
- AliExpress
- Робототехнические магазины

### Generic режим (для экспериментов)

Для лидаров с неизвестным протоколом (например, MOP3):
- Автоматическое определение формата данных
- Экспериментальный режим
- Возможность добавления специфичных драйверов

## Установка

### Зависимости

```bash
pip install pyserial
```

### Для RPLidar

Никаких дополнительных библиотек не требуется - используется стандартный протокол.

## Использование

### 1. Подключение лидара

**RPLidar:**
- Подключите через USB
- Проверьте порт: `ls -l /dev/ttyUSB*` или `/dev/ttyACM*`
- Добавьте пользователя в группу dialout:
  ```bash
  sudo usermod -a -G dialout $USER
  # Перелогиньтесь
  ```

### 2. Запуск узла

**Для RPLidar:**
```bash
ros2 run watchdog_lidar lidar_node --ros-args \
  -p lidar.type:=rplidar \
  -p lidar.port:=/dev/ttyUSB0 \
  -p lidar.baudrate:=115200
```

**Для RPLidar A3:**
```bash
ros2 run watchdog_lidar lidar_node --ros-args \
  -p lidar.type:=rplidar \
  -p lidar.port:=/dev/ttyUSB0 \
  -p lidar.baudrate:=256000
```

**Для экспериментов с MOP3 или другими:**
```bash
ros2 run watchdog_lidar lidar_node --ros-args \
  -p lidar.type:=generic \
  -p lidar.port:=/dev/ttyUSB0 \
  -p lidar.baudrate:=115200
```

### 3. Проверка работы

```bash
# Просмотр сканов лидара
ros2 topic echo /sensor/lidar/scan

# Проверка статуса
ros2 topic echo /lidar/status

# Визуализация (если установлен RViz2)
rviz2
# Добавьте LaserScan topic: /sensor/lidar/scan
```

## Работа с MOP3 лидаром

Если у вас есть лидар от Xiaomi MOP3, можно попробовать generic режим:

```bash
ros2 run watchdog_lidar lidar_node --ros-args \
  -p lidar.type:=generic \
  -p lidar.port:=/dev/ttyUSB0 \
  -p lidar.baudrate:=115200
```

**Примечание:** Для полной поддержки MOP3 потребуется:
1. Исследование протокола (анализ данных)
2. Возможно реверс-инжиниринг
3. Создание специфичного драйвера

Можно помочь с анализом протокола, если предоставите образцы данных.

## Параметры

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `lidar.type` | string | `rplidar` | Тип лидара (rplidar, generic) |
| `lidar.port` | string | `/dev/ttyUSB0` | Последовательный порт |
| `lidar.baudrate` | int | `115200` | Скорость передачи |
| `lidar.frame_id` | string | `lidar_frame` | Имя фрейма для TF |
| `lidar.angle_min` | double | `-3.14159` | Минимальный угол (рад) |
| `lidar.angle_max` | double | `3.14159` | Максимальный угол (рад) |
| `lidar.range_min` | double | `0.05` | Минимальная дальность (м) |
| `lidar.range_max` | double | `12.0` | Максимальная дальность (м) |
| `lidar.scan_rate` | double | `10.0` | Частота публикации (Гц) |

## Топики

### Публикации

- `/sensor/lidar/scan` (sensor_msgs/LaserScan) - Данные сканирования
- `/lidar/status` (std_msgs/String) - Статус лидара

## Интеграция с навигацией

Данные лидара используются для:

1. **SLAM (SLAM)** - построение карты и локализация
2. **Obstacle avoidance** - избежание препятствий
3. **Navigation** - планирование пути

Пример использования в навигации:
```python
from sensor_msgs.msg import LaserScan

def scan_callback(self, msg: LaserScan):
    # Проверяем препятствия впереди
    front_ranges = msg.ranges[len(msg.ranges)//4:3*len(msg.ranges)//4]
    min_distance = min([r for r in front_ranges if r > msg.range_min])
    
    if min_distance < 0.3:  # 30 см до препятствия
        # Останавливаемся или объезжаем
        pass
```

## Отладка

### Проблема: "Порт не найден"

```bash
# Проверьте подключение
ls -l /dev/ttyUSB* /dev/ttyACM*

# Проверьте права
sudo chmod 666 /dev/ttyUSB0

# Добавьте в группу dialout
sudo usermod -a -G dialout $USER
```

### Проблема: "Нет данных от лидара"

1. Проверьте подключение кабеля
2. Проверьте питание лидара (обычно 5V)
3. Попробуйте другую скорость передачи
4. Проверьте логи: `ros2 topic echo /lidar/status`

### Проблема: "Неправильные данные"

1. Проверьте тип лидара в параметрах
2. Проверьте скорость передачи (baudrate)
3. Для generic режима - возможно нужна настройка протокола

## Расширение функционала

### Добавление нового драйвера

1. Создайте класс наследующий `LidarDriver`
2. Реализуйте методы: `connect`, `disconnect`, `start_scanning`, `stop_scanning`, `get_scan`
3. Добавьте инициализацию в `lidar_node.py`

Пример:
```python
class CustomLidarDriver(LidarDriver):
    def connect(self) -> bool:
        # Ваша реализация
        pass
    # ... остальные методы
```
