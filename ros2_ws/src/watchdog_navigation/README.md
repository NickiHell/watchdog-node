# watchdog_navigation

ROS2 пакет для навигации с построением карты на основе лидара.

## Функции

- **Построение карты (SLAM)** - автоматическое построение карты из данных лидара
- **Локализация** - определение позиции робота на карте
- **Планирование пути** - поиск оптимального пути к цели
- **Избежание препятствий** - обход препятствий в реальном времени
- **Интеграция с лидаром** - использование данных лидара для навигации

## Требования

### SLAM (опционально, но рекомендуется)

Для построения карты используется `slam_toolbox`:

```bash
sudo apt install ros-humble-slam-toolbox
```

Или установка из исходников для последней версии.

## Использование

### 1. Запуск SLAM узла (для построения карты)

```bash
ros2 run slam_toolbox async_slam_toolbox_node \
  --ros-args \
  -p slam_toolbox.odom_frame:=odom \
  -p slam_toolbox.map_frame:=map \
  -p slam_toolbox.base_frame:=base_link \
  -p slam_toolbox.scan_topic:=/sensor/lidar/scan
```

### 2. Запуск узла навигации

```bash
ros2 run watchdog_navigation navigation_node --ros-args \
  -p use_slam:=true \
  -p safety_distance:=0.3 \
  -p max_linear_velocity:=0.5
```

### 3. Отправка цели

```bash
ros2 topic pub /navigation/goal geometry_msgs/msg/PoseStamped \
  "{header: {frame_id: 'map'}, pose: {position: {x: 2.0, y: 1.0}}}"
```

### 4. Проверка статуса

```bash
ros2 topic echo /navigation/status
```

## Параметры

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `safety_distance` | double | `0.3` | Безопасное расстояние до препятствий (м) |
| `max_linear_velocity` | double | `0.5` | Максимальная линейная скорость (м/с) |
| `max_angular_velocity` | double | `1.0` | Максимальная угловая скорость (рад/с) |
| `goal_tolerance` | double | `0.1` | Точность достижения цели (м) |
| `use_slam` | bool | `true` | Использовать SLAM для карты |
| `inflation_radius` | double | `0.3` | Радиус раздувания препятствий (м) |

## Топики

### Подписки

- `/sensor/lidar/scan` (sensor_msgs/LaserScan) - Данные лидара
- `/navigation/goal` (geometry_msgs/PoseStamped) - Целевая позиция
- `/navigation/cancel` (std_msgs/String) - Отмена навигации
- `/map` (nav_msgs/OccupancyGrid) - Карта (от SLAM)
- `/pose` (geometry_msgs/PoseWithCovarianceStamped) - Позиция робота

### Публикации

- `/cmd_vel` (geometry_msgs/Twist) - Команды движения
- `/navigation/status` (std_msgs/String) - Статус навигации

## Режимы работы

### Режим с SLAM (рекомендуется)

1. **Построение карты**: Лидар сканирует пространство, SLAM строит карту
2. **Локализация**: Робот определяет свою позицию на карте
3. **Планирование пути**: Путь строится на основе карты
4. **Движение**: Робот следует по пути, объезжая препятствия

### Простой режим (без SLAM)

- Движение напрямую к цели
- Избежание препятствий на основе лидара
- Без построения карты

## Построение карты

### Автоматическое построение

При запуске с `use_slam:=true` карта строится автоматически:
1. Робот получает данные от лидара
2. SLAM узел обрабатывает данные и строит карту
3. Карта публикуется в топик `/map`
4. Навигационный узел использует карту для планирования

### Сохранение карты

Сохранить карту можно через slam_toolbox:
```bash
ros2 service call /slam_toolbox/save_map slam_toolbox/srv/SaveMap "{name: {data: '/path/to/map'}}"
```

### Загрузка карты

Для локализации на известной карте:
```bash
ros2 run slam_toolbox localization_slam_toolbox_node \
  --ros-args \
  -p slam_toolbox.odom_frame:=odom \
  -p slam_toolbox.map_frame:=map \
  -p slam_toolbox.scan_topic:=/sensor/lidar/scan \
  -p slam_toolbox.map_file_name:="/path/to/map.pgm"
```

## Интеграция с другими модулями

### Использование в контроллере

```python
# Отправка цели от пользователя
goal = PoseStamped()
goal.header.frame_id = 'map'
goal.pose.position.x = x
goal.pose.position.y = y
self.goal_pub.publish(goal)
```

### Комбинация с голосовыми командами

Можно добавить голосовые команды для навигации:
- "иди в комнату" → вычисление цели на карте
- "остановись" → отмена навигации

## Отладка

### Проблема: "Карта не строится"

1. Проверьте, запущен ли slam_toolbox узел
2. Проверьте данные лидара: `ros2 topic echo /sensor/lidar/scan`
3. Проверьте odometry: `ros2 topic echo /odom`
4. Убедитесь, что робот движется (для построения карты)

### Проблема: "Путь не найден"

1. Проверьте, что карта построена
2. Убедитесь, что цель в свободном пространстве
3. Попробуйте уменьшить `inflation_radius`
4. Проверьте, что стартовая позиция валидна

### Проблема: "Робот не достигает цели"

1. Проверьте `goal_tolerance` - может быть слишком строгим
2. Убедитесь, что цель достижима (нет препятствий)
3. Проверьте точность локализации

## Производительность

**Рекомендации:**

- Для Raspberry Pi: используйте упрощенный режим без SLAM или снизьте частоту обновления
- Для более мощных систем: полный SLAM с высокой частотой
- Разрешение карты: 0.05м (5см) - хороший баланс точности и производительности
