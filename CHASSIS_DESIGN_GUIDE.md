# Руководство по проектированию шасси и корпуса

## Требования к размерам

### Рекомендуемые размеры корпуса:
- **Длина:** 300-350мм
- **Ширина:** 250-300мм
- **Высота:** 150-200мм (без лидара)
- **Высота с лидаром:** +70-100мм (лидар сверху)

## Размещение компонентов

### Нижний уровень (шасси):
- **Моторы 25GA370** (2 шт) - по бокам, крепление на раме
- **Колеса** - диаметр 60-80мм (зависит от мотора)
- **Аккумулятор** - в центре или сзади, низко для стабильности
  - Размеры: ~110×55×35мм (для 5000mAh)
  - Или: ~90×45×25мм (для 2200mAh)

### Средний уровень:
- **STM32 Nucleo F411RE** (~70×52×18мм)
  - Крепление на стойках или платформе
  - Доступ к USB для программирования
  - Доступ к UART пинам для связи с Beelink
  
- **DRV8833 драйверы** (2 шт, ~25×20×6мм каждый)
  - Рядом с моторами для коротких проводов
  - Обеспечить вентиляцию
  
- **Преобразователи напряжения:**
  - LM2596S (2 шт) - для 5V
  - Преобразователь 11.1V→3.3V - для STM32
  - Преобразователь для Beelink - рядом с Beelink

### Верхний уровень:
- **Beelink Mini PC** (~115×102×41мм)
  - Центральное расположение
  - **Обязательно вентиляция** (вентиляционные отверстия или вентилятор)
  - Доступ к USB портам со всех сторон
  - Доступ к разъему питания
  
- **Крепление лидара:**
  - Высота башни: ~70мм от верха Beelink
  - Центральное расположение для 360° обзора
  - Стабильное крепление (вибрации ухудшают качество)

- **Камера:**
  - Крепление на передней части или на башне
  - Угол обзора для распознавания лиц

## Важные моменты при проектировании

### 1. Вентиляция
- **Beelink Mini PC** - обязательна вентиляция
  - Вентиляционные отверстия сверху и сбоку
  - Или активное охлаждение (вентилятор)
  - Зазор минимум 5-10мм вокруг устройства

### 2. Центр тяжести
- Аккумулятор внизу для стабильности
- Beelink и лидар сверху - центр тяжести не должен быть слишком высоким
- Ширина колесной базы: минимум 200мм для устойчивости

### 3. Доступ к разъемам
- **Beelink Mini PC:**
  - USB порты (камера, лидар, микрофон)
  - Разъем питания
  - Возможно HDMI для отладки
  
- **STM32 Nucleo:**
  - USB для программирования
  - UART пины (PA9, PA10) для связи с Beelink
  - ST-Link разъем

### 4. Кабели и проводка
- Организация кабелей (стяжки, каналы)
- Избегать перегибов и натяжения
- Разделение силовых и сигнальных кабелей
- Защита от вибраций

### 5. Крепление лидара
- Стабильное крепление (вибрации = плохие данные)
- Центральное расположение
- Высота: минимум 70мм от верха корпуса
- Возможность регулировки уровня

### 6. Крепление камеры
- Угол обзора для распознавания лиц
- Возможность поворота/наклона
- Защита от вибраций

## Материалы

### Рекомендуемые:
1. **Алюминий** (2-3мм)
   - Прочность, легкость
   - Хорошая теплопроводность
   - Легко обрабатывать

2. **Оргстекло** (3-5мм)
   - Легче алюминия
   - Прозрачное (видно компоненты)
   - Легко обрабатывать

3. **3D печать** (PLA/PETG)
   - Гибкость дизайна
   - Легко создавать сложные формы
   - Можно печатать крепления для компонентов

### Крепеж:
- Винты M3 (основной размер)
- Стойки для многоуровневого размещения
- Гайки, шайбы

## Рекомендации по проектированию

1. **Начните с нижнего уровня** - моторы и колеса
2. **Определите колесную базу** - расстояние между осями колес (рекомендуется 200-250мм)
3. **Разместите аккумулятор** - низко и по центру для стабильности
4. **Средний уровень** - STM32 и драйверы
5. **Верхний уровень** - Beelink с вентиляцией
6. **Лидар сверху** - центрально, стабильно

## Файлы для справки

В проекте есть 3D модели в папке `3d_models/`:
- FreeCAD скрипты для создания корпуса
- Параметры можно изменить в `3d_models/freecad_scripts/parameters.py`

## Проверка перед изготовлением

- [ ] Все компоненты помещаются
- [ ] Есть вентиляция для Beelink
- [ ] Доступ к разъемам USB
- [ ] Доступ к UART пинам STM32
- [ ] Центр тяжести не слишком высокий
- [ ] Колесная база достаточна для устойчивости
- [ ] Кабели организованы
- [ ] Лидар крепится стабильно
